ما هي لغة البرمجة؟ النظرية الأساسية قبل ما نكتب سطر كود

المُجمَّعة مقابل المُفسَّرة

لغات مُجمَّعة (Compiled)
تُحوَّل الشفرة كاملةً إلى ملف تنفيذي قبل التشغيل. النتيجة: سرعة عالية جداً. مثال: C++، Rust، Go
لغات مُفسَّرة (Interpreted)
تُقرأ وتُنفَّذ سطراً بسطر عبر مفسِّر. مرونة أكبر وتطوير أسرع. مثال: Python، JavaScript، Ruby

الكتابة الساكنة مقابل الديناميكية

في الكتابة الساكنة (Static Typing) يجب تحديد نوع كل متغير وقت كتابة الكود. يكتشف المُجمِّع الأخطاء قبل تشغيل البرنامج. C++ ساكنة النوع.

في الكتابة الديناميكية (Dynamic Typing) يُحدَّد نوع المتغير أثناء التشغيل. أكثر مرونة لكن الأخطاء تظهر متأخراً. Python ديناميكية النوع.

INFO
في C++ تكتب: int age = 25; — النوع int مُحدَّد ولا يتغير.
في Python تكتب: age = 25 — المُفسِّر يكتشف النوع تلقائياً.

الكتابة القوية مقابل الضعيفة

كتابة قوية (Strong)
لا يمكن خلط الأنواع دون تحويل صريح. لا تجمع رقم مع نص تلقائياً. C++ قوية الكتابة.
كتابة ضعيفة (Weak)
تسمح ببعض التحويلات التلقائية. في JavaScript: '5' + 3 تعطي '53' بدلاً من خطأ.

البرمجة الإجرائية والكائنية والدوالية

  • الإجرائية (Procedural): الكود عبارة عن سلسلة خطوات تُنفَّذ بالترتيب. نبدأ بها في هذا المقرر.
  • كائنية التوجه (OOP): نُنظِّم الكود في وحدات تُسمَّى كائنات تجمع البيانات والسلوك معاً.
  • الدوالية (Functional): نعامل الدوال كقيم يمكن تمريرها. متقدمة — نذكرها مرجعاً فقط.
TIP
C++ تدعم الثلاثة. هاد يعني إنك رح تتعلم التفكير الإجرائي أول، ثم الكائني — وبعد ما تتعلمهم رح تفهم ليش اخترع الناس كل طريقة.

ماذا يحتاج برنامج C++ ليعمل؟

على عكس Python التي تُفسَّر مباشرة، كود C++ يجب أن يُحوَّل إلى ملف تنفيذي أولاً. هذه العملية تحتاج أداتين أساسيتين:

⚙️
المُجمِّع — GCC
GNU Compiler Collection

الأداة التي تُحوِّل كودك المكتوب بـ C++ إلى ملف تنفيذي يفهمه الكمبيوتر. بدونه لا يوجد برنامج.

  • مجاني ومفتوح المصدر
  • يعمل على Windows / Mac / Linux
  • يُستدعى عبر الطرفية (Terminal)
تحقق من التثبيت
g++ --version
🔵
المحرر — VS Code
Visual Studio Code

محرر النصوص الذي تكتب فيه كودك. اخترنا VS Code لأنه الأكثر استخداماً في العالم وهو مجاني تماماً.

  • واجهة سهلة وألوان تساعد على القراءة
  • يدعم C++ مع إضافة C/C++ Extension
  • يحتوي على طرفية مدمجة (Terminal)
تحميله من
code.visualstudio.com

🖥️ الطرفية (Terminal) — ما هي؟

الطرفية هي نافذة نصية تتحكم فيها بالكمبيوتر عبر أوامر مكتوبة. في VS Code الأزرق، افتح الطرفية المدمجة بالضغط على Ctrl + ` أو من القائمة: View → Terminal.

أوامر التنقل الأساسية

ls اعرض محتويات المجلد

يطبع أسماء جميع الملفات والمجلدات الموجودة في موقعك الحالي.

مثال
$ ls Desktop Documents main.cpp projects

💡 استخدمه لتعرف أين أنت وماذا يوجد حولك.

cd انتقل إلى مجلد آخر

يغيّر موقعك الحالي (Change Directory). أنت دائماً "داخل" مجلد ما.

أمثلة
$ cd projects # ادخل مجلد projects $ cd .. # ارجع مجلداً للخلف $ cd ~ # اذهب للمجلد الرئيسي

كيف تفتح VS Code من الطرفية؟

بعد ما تنتقل إلى مجلد مشروعك بـ cd، اكتب:

$ code .

النقطة تعني "المجلد الحالي". VS Code سيفتح مباشرة على مجلدك — وتجد ملفاتك في Explorer على اليسار.

تجميع وتشغيل برنامج C++

بعد ما تكتب كودك في VS Code واحفظته (Ctrl+S)، افتح الطرفية المدمجة ونفّذ هذين الأمرين بالترتيب:

١
التجميع — حوِّل الكود إلى برنامج
$ g++ main.cpp -o main
g++ استدعاء مُجمِّع GCC لـ C++ main.cpp اسم ملف الكود المصدري
-o حدِّد اسم الملف الناتج main اسم البرنامج التنفيذي الناتج

إذا لم تظهر أي رسالة = نجح التجميع ✅. إذا ظهرت أخطاء = راجع الكود ❌

٢
التشغيل — نفِّذ البرنامج
$ ./main

./ تعني "ابحث عن البرنامج في المجلد الحالي". على Windows يكون الأمر: .\main.exe

مثال كامل — من الصفر حتى تشغيل البرنامج:
جلسة طرفية كاملة
# ١. انتقل إلى مجلد مشروعك $ cd ~/Desktop/my-cpp-project # ٢. افتح VS Code في هذا المجلد $ code . # ٣. بعد ما كتبت وحفظت main.cpp — جمِّعه $ g++ main.cpp -o main # ٤. شغِّل البرنامج $ ./main مرحباً بالعالم
NOTE
خطأ شائع: إذا كتبت g++ main.cpp -o -o بدل g++ main.cpp -o main سيعطيك خطأ. تأكد أن -o يأتي بعدها مباشرةً اسم الملف الناتج.

بنية برنامج C++ الأساسية

أول برنامج في C++
#include <iostream> using namespace std; int main() { cout << "مرحباً بالعالم" << endl; return 0; }
  • #include <iostream> — يُدرج مكتبة الإدخال والإخراج الأساسية.
  • using namespace std; — يتيح استخدام cout و cin مباشرة.
  • int main() — نقطة البداية لكل برنامج C++.
  • return 0; — تعني أن البرنامج انتهى بنجاح.

1) برنامج ترحيب تفاعلي

✏️ اكتب برنامجاً C++ يطلب من المستخدم إدخال اسمه (من نوع string) ثم عمره (من نوع int)، وبعد ذلك يطبع رسالة ترحيبية مخصصة مثل:

مثال الإخراج المتوقع
مرحباً أحمد، عمرك 22 سنة. أهلاً بك في عالم البرمجة! 🚀

المطلوب: استخدم cin و cout بشكل صحيح، وتأكد من التعامل مع المسافات في الاسم (يمكنك استخدام getline إذا أردت التحدي).

2) استكشاف أنواع البيانات الأساسية

🔢 عرّف متغيرات تمثل الأنواع التالية: int, float, double, char, bool, string. قم بإسناد قيم مناسبة لكل منها (مثل: 42, 3.14f, 2.71828, 'Z', true, "C++ قوية"). ثم اطبع جميع القيم باستخدام cout مع رسالة واضحة لكل متغير.

ملاحظة: عند طباعة قيمة bool ستظهر 1 (للـ true) أو 0 (للـ false). جرّب تغيير القيمة وشاهد الناتج.

🧪 مثال جزئي: cout << "قيمة العدد الصحيح: " << myInt << endl;
3) الآلة الحاسبة البسيطة (عددان صحيحان)

🧮 اكتب برنامجاً يطلب من المستخدم إدخال عددين صحيحين (int)، ثم يقوم بحساب وطباعة ما يلي:

  • المجموع (+)
  • الفرق (الأول - الثاني)
  • الضرب (*)
  • القسمة الصحيحة (ناتج قسمة العدد الأول على الثاني) مع الانتباه إلى مشكلة القسمة الصحيحة
  • باقي القسمة (%)

⚠️ تحدٍ إضافي: قم بحساب القسمة العشرية الدقيقة (double) عن طريق تحويل أحد العددين إلى double.

📌 تذكر: 7 / 2 = 3 (لأنها int)، بينما (double)7 / 2 = 3.5
4) تحويل درجة الحرارة (مئوية ← فهرنهايت)

🌡️ اكتب برنامجاً يقرأ درجة الحرارة بالدرجة المئوية (double celsius) ثم يحولها إلى فهرنهايت وفق المعادلة: F = C × 9/5 + 32. أظهر الناتج بمنتهى الدقة (مثلاً: "درجة الحرارة بالفهرنهايت: 77.00").

💡 ملاحظات ومساعدات إضافية

  • لكل برنامج، ابدأ بقالب: #include <iostream> using namespace std; int main() { ... return 0; }
  • جرب استخدام endl أو "\n" للسطر الجديد.
  • في التمارين التي تتطلب مقارنات، تذكر أن العوامل المنطقية تعطي قيمة bool (true/false).

جملة if / else if / else

if / else if / else
int score = 85; if (score >= 90) { cout << "ممتاز"; } else if (score >= 75) { cout << "جيد جداً"; } else if (score >= 60) { cout << "جيد"; } else { cout << "راسب"; }

الحلقات الثلاث

الحلقات المسماة (Labeled Loops) & جملة switch
استخدام التصنيفات مع break/continue · التبديل بين عدة مسارات

🏷️ تسمية الحلقات (Naming a Loop)

حلقة مسماة — الخروج من الحلقات المتداخلة
// تسمية الحلقة الخارجية outer: for (int i = 1; i <= 3; i++) { for (int j = 1; j <= 3; j++) { if (i * j == 4) { cout << "خرجت من الحلقتين عند " << i << "," << j << endl; break outer; // يخرج من outer وليس فقط الداخلية } cout << i << "," << j << " "; } } // continue مع تسمية الحلقة start: while (int count = 0; count < 5; count++) { if (count % 2 == 0) continue start; // ينتقل إلى التكرار التالي من الحلقة المسماة start cout << count << " "; }
  • التسمية (label): تُكتب قبل الحلقة مباشرة متبوعة بنقطتين رأسيتين outer:.
  • break label; — يخرج من الحلقة المحددة (حتى لو كانت خارجية في حلقات متداخلة).
  • continue label; — يتخطى بقية التكرار الحالي للحلقة المسماة ويبدأ التكرار التالي فيها.
  • فائدة أساسية: التحكم الدقيق في الحلقات المتداخلة دون الحاجة إلى متغيرات إضافية.

🔀 جملة switch — متعدد الفروع

switch / case / default
char grade = 'B'; switch (grade) { case 'A': cout << "ممتاز 🔥"; break; case 'B': cout << "جيد جداً 👍"; break; case 'C': cout << "جيد ✅"; break; case 'D': case 'F': // نفس السلوك لـ D و F cout << "راسب ⚠️"; break; default: cout << "مدخل غير صالح"; }
⚠️ break في switch
بدون break يستمر التنفيذ إلى الـ case التالي (fall‑through). هذا قد يكون مقصوداً أحياناً.
🧪 default
ينفذ إذا لم يطابق أي case — اختياري لكن يُنصح به دائماً.
NOTE
الفرق بين if المتعددة و switch — switch تصلح عند الاختبار على قيمة واحدة (عدد صحيح، حرف، تعداد) وتكون أسرع وأوضح من if-else if المتسلسل. أما if فتناسب الشروط المعقدة (مقارنات، نطاقات، عمليات منطقية).

📘 ٤ تمارين لتعزيز المفاهيم

1) آلة حاسبة بسيطة باستخدام switch

🧮 اكتب برنامجاً يطلب من المستخدم إدخال رقمين (double a, b) ثم حرف العملية (+ , - , * , /). استخدم switch لتنفيذ العملية المطلوبة وطباعة النتيجة. تأكد من:

  • معالجة حالة القسمة على صفر (إذا كانت العملية / و b == 0 اطبع خطأ).
  • استخدام default لحالة إدخال عملية غير صالحة.
  • جرب مفهوم fall-through لتوحيد سلوك العمليات المتشابهة (اختياري).
🧪 مثال: الإدخال: 10 5 + → الناتج: 10 + 5 = 15
2) قائمة تفاعلية باستخدام do-while و switch

📋 اكتب برنامجاً يعرض قائمة للمستخدم كالتالي:

القائمة
1. طباعة "Hello World!" 2. طباعة التاريخ الحالي (يمكنك طباعة نص وهمي) 3. التحقق من زوجية رقم يدخله المستخدم 4. الخروج من البرنامج

استخدم حلقة do-while لتكرار عرض القائمة حتى يختار المستخدم الخيار (4). داخل الحلقة، استخدم switch لتنفيذ الإجراء المناسب لكل خيار. تأكد من التعامل مع الإدخال غير الصالح باستخدام default.

⏳ ملاحظة: في الخيار رقم 2، يمكنك طباعة أي نص مثل "2025-04-07". الهدف هو التدرب على switch داخل حلقة.

Bubble Sort — فرز الفقاعة خطوة بخطوة

فكرة الفرز بالفقاعة: نمشي على المصفوفة مراراً، وفي كل مرور نقارن كل عنصرين متجاورين ونبدّلهما إذا كانا في الترتيب الخاطئ. الأرقام الكبيرة "تطفو" للأعلى مثل الفقاعة.

Bubble Sort — فرز تصاعدي
int arr[] = {64, 34, 25, 12, 22}; int n = 5; for (int i = 0; i < n-1; i++) { for (int j = 0; j < n-i-1; j++) { if (arr[j] > arr[j+1]) { // تبادل المكانين int temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp; } } } // النتيجة: {12, 22, 25, 34, 64}

المصفوفات ثنائية الأبعاد — الجدول الحي

المصفوفة ثنائية الأبعاد هي في الحقيقة "مصفوفة من المصفوفات" — فكّر فيها كجدول بصفوف وأعمدة. مثالها الكلاسيكي: جدول الدرجات حيث كل صف طالب وكل عمود مادة.

// grades[طالب][مادة] int grades[3][3] = { {90, 85, 78}, // أحمد: رياضيات، علوم، لغة {70, 92, 88}, // سارة {95, 60, 74} // خالد }; // طباعة معدل كل طالب for (int i = 0; i < 3; i++) { int sum = 0; for (int j = 0; j < 3; j++) sum += grades[i][j]; cout << "معدل الطالب " << i << ": " << sum/3 << endl; }

الفئة هي القالب أو المخطط. الكائن هو النسخة الفعلية منه. مثال: السيارة فئة، وسيارتك الحمراء كائن من تلك الفئة.

class Entity { // الفئة الأم public: float x, y; int health; virtual void update() {} }; class Player : public Entity { public: void update() override { // تحرك بالكيبورد } }; class Enemy : public Entity { public: void update() override { // تحرك نحو اللاعب } }; // تعددية الأشكال في العمل vector<Entity*> entities; entities.push_back(new Player()); entities.push_back(new Enemy()); for (Entity* e : entities) { e->update(); // كل كائن ينفّذ update() الخاصة به }

public و private — ليستا مجرد كلمات

التغليف (Encapsulation) يعني إخفاء التفاصيل الداخلية وتقديم واجهة نظيفة للعالم الخارجي. تخيّل حسابك البنكي: يمكنك الإيداع والسحب، لكنك لا تستطيع تعديل الرصيد مباشرة — الفئة هي المسؤولة عن حمايته.

BankAccount — التغليف الحقيقي
class BankAccount { private: // لا أحد يلمسها مباشرة — حمايتها واجب double balance; string ownerName; public: // Constructor — يُنفَّذ عند إنشاء الكائن BankAccount(string name, double initial) { ownerName = name; balance = initial; } // Destructor — يُنفَّذ تلقائياً عند تدمير الكائن ~BankAccount() { cout << ownerName << ": الحساب مُغلَق" << endl; } void deposit(double amount) { if (amount > 0) balance += amount; } bool withdraw(double amount) { if (amount > balance) return false; // رفض balance -= amount; return true; } double getBalance() const { return balance; } }; // الاستخدام BankAccount acc("أحمد", 1000); acc.deposit(500); acc.withdraw(200); // acc.balance = 9999; ← خطأ في التصريف! محمي
Constructor دالة البناء
تُنفَّذ تلقائياً لحظة إنشاء الكائن. اسمها = اسم الفئة. لا تُرجع قيمة. تُهيّئ البيانات في حالة آمنة.
Destructor دالة الهدم
تُنفَّذ تلقائياً لحظة تدمير الكائن (انتهاء النطاق أو delete). مسبوقة بـ ~. مكانها المثالي لتحرير الذاكرة.

مؤشر this — "أنا نفسي"

داخل كل دالة عضو في الفئة يتوفر مؤشر سري اسمه this يشير إلى الكائن الحالي نفسه. نستخدمه حين يتشابه اسم المعامل مع اسم العضو، أو حين نريد إرجاع الكائن نفسه (نمط method chaining).

استخدام this لتجنب التشابه + method chaining
class Counter { private: int count; public: Counter(int count) { this->count = count; // this يفرّق بين العضو والمعامل } // يُرجع الكائن نفسه → نستطيع التسلسل Counter& increment() { count++; return *this; } Counter& add(int n) { count+=n; return *this; } void print() { cout << count << endl; } }; Counter c(0); c.increment().increment().add(5).print(); // 7
INFO
متى تستخدم OOP؟ حين يكون لديك شيء له حالة (state) وسلوك (behaviour) معاً — لاعب في لعبة، حساب بنكي، ملف على القرص. البيانات المجردة التي لا تملك سلوكاً قد تكون struct أبسط وأوضح.

١ محلل الأرقام السرية 🔍 مصفوفات · حلقات

التحدي: لديك مصفوفة من الأعداد الصحيحة. اكتب برنامجاً يطبع فقط الأعداد الأولية (Prime) منها، ثم يطبع مجموعها. العدد الأولي لا يقبل القسمة إلا على ١ وعلى نفسه.

الحل النموذجي
bool isPrime(int n) { if (n < 2) return false; for (int i = 2; i * i <= n; i++) // نفحص حتى جذر n فقط if (n % i == 0) return false; return true; } int nums[] = {2,9,17,4,23,15,11,6}; int n=8, sumPrimes=0; cout << "الأعداد الأولية: "; for(int i=0;i<n;i++) if(isPrime(nums[i])) { cout<<nums[i]<<" "; sumPrimes+=nums[i]; } cout << endl << "مجموعها: " << sumPrimes; // الأعداد الأولية: 2 17 23 11 مجموعها: 53
💡 لماذا i*i ≤ n؟ لأن كل عامل أكبر من √n يقابله عامل أصغر منه — فحص√n يكفي لاكتشاف جميع العوامل.
٢ غربال الحروف المتحركة 🔤 string · char · حلقات

التحدي: أعطِ المستخدم كلمة إنجليزية، واطبع الحروف المتحركة (vowels) فيها بالترتيب، وعدّها. الحروف المتحركة هي: a e i o u (كبيرة وصغيرة).

الحل النموذجي
string word = "Programming"; string vowels = "aeiouAEIOU"; int count = 0; cout << "الحروف المتحركة في "" << word << "": "; for (char ch : word) { // range-based for if (vowels.find(ch) != string::npos) { cout << ch << " "; count++; } } cout << "(" << count << " حروف)" << endl; // o a i (3 حروف)
💡 تحدٍّ إضافي: عدّل الكود ليعدّ الحروف الساكنة أيضاً، ثم اطبع النسبة المئوية لكل منهما.

التحدي: صمّم فئة Student تحمل اسم الطالب وقائمة درجاته. أضف دوال: addGrade() ،getAverage() ،getLetterGrade() (A/B/C/D/F).

الحل النموذجي
class Student { private: string name; vector<int> grades; public: Student(string n) : name(n) {} void addGrade(int g) { grades.push_back(g); } double getAverage() { if(grades.empty()) return 0; int sum=0; for(int g : grades) sum+=g; return (double)sum / grades.size(); } string getLetterGrade() { double avg = getAverage(); if(avg>=90) return "A"; if(avg>=80) return "B"; if(avg>=70) return "C"; if(avg>=60) return "D"; return "F"; } void print() { cout<<name<<" | معدل: "<<getAverage() <<" | تقدير: "<<getLetterGrade()<<endl; } };
٤ حارس المصفوفة — بلا تكرار! 🛡️ vector · خوارزميات

التحدي: لديك مصفوفة فيها أرقام مكررة. اكتب دالة تُرجع vector جديداً يحتوي فقط على الأرقام الفريدة (بدون تكرار) محافظةً على ترتيب الظهور الأول.

الحل النموذجي
vector<int> removeDuplicates(vector<int> arr) { vector<int> result; for (int x : arr) { bool found = false; for (int r : result) if (r == x) { found = true; break; } if (!found) result.push_back(x); } return result; } // {5,3,5,1,3,9,1} → {5,3,1,9}
💡 تحدٍّ إضافي: عدّل الدالة لتطبع أيضاً كم مرة تكرر كل رقم.
